# 要件定義書

## 1. 目的
- 現場判断を支えるナレッジの集約と再利用
- 承認・配信プロセスの標準化と可視化

## 2. ステークホルダー
- 施工管理 (主利用者)
- 品質保証室
- 安全衛生室
- 技術本部
- 本社管理部門
- 協力会社 (閲覧のみ)

## 3. 前提・制約
- ブラウザベースの利用を前提
- 現場の通信環境は安定しない場合がある
- 法令/SOPは改訂履歴を保持する

## 4. 要件一覧 (抜粋)
| ID | 要件 | 優先度 |
| --- | --- | --- |
| FR-01 | ナレッジを種別別に検索・閲覧できる | High |
| FR-02 | 承認フローを可視化し、ステータスを更新できる | High |
| FR-03 | SOP/法令/事故レポート/相談を統合表示できる | High |
| FR-04 | 役割に応じた閲覧・編集権限を制御できる | High |
| FR-05 | 重要ナレッジの配信通知を行える | Medium |
| NFR-01 | 主要画面は3秒以内で表示される | High |
| NFR-02 | 監査ログを7年間保持する | Medium |

## 5. 非機能要件サマリ
- 性能: 同時利用 300名を想定
- 可用性: 月間稼働率 99.5%以上
- セキュリティ: 役割ベースアクセス制御、監査ログ

## 6. 受入条件 (概要)
- 主要ユースケースの操作が要件どおり実行できる
- 権限設定が想定通りに機能する
- 監査ログと配信履歴が追跡可能である
