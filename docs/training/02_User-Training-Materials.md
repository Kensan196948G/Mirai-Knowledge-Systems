## 1. システム概要

### 1.1 Mirai Knowledge Systemsとは

建設土木現場での迅速な判断を支援するナレッジマネジメントシステム

**できること**:
- 📋 SOP（標準施工手順）の検索・閲覧
- 📜 法令・規格の確認
- ⚠️ 過去の事故レポート参照
- 💬 専門家への相談
- 📝 現場知見の共有

### 1.2 なぜ重要か

```
┌─────────────────────────────────────────────┐
│  従来の課題                                  │
│  ・経験者の知識が属人化                      │
│  ・過去事例の検索に時間がかかる              │
│  ・SOPが更新されても周知されない             │
└─────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  MKSで解決                                   │
│  ・知識をシステムで共有・蓄積               │
│  ・必要な情報を瞬時に検索                   │
│  ・更新情報が自動通知される                 │
└─────────────────────────────────────────────┘
```

### 1.3 対象ユーザーと権限

| 役割 | できること |
|------|-----------|
| 施工管理 | 検索、作成、相談投稿 |
| 安全衛生担当 | 検索、作成、事故レポート作成 |
| 技術専門家 | 検索、作成、相談回答 |
| 閲覧者 | 公開ナレッジの閲覧のみ |

## 2. ログインと画面構成

### 2.1 ログイン方法

**手順**:
1. ブラウザで `https://knowledge.example.com` にアクセス
2. ユーザー名・パスワードを入力
3. 「ログイン」ボタンをクリック

**初回ログイン時**:
- パスワード変更が必要
- 8文字以上、英大文字・小文字・数字・記号を含む

### 2.2 ログインできない場合

チェックリスト:
- [ ] ユーザー名は正しいか（大文字・小文字を区別）
- [ ] Caps Lockがオフになっているか
- [ ] 5回失敗でロック → 30分待つか管理者に連絡

### 2.3 画面構成

```
┌────────────────────────────────────────────────────┐
│  [ロゴ]  [     検索バー     ]  [通知🔔] [ユーザー▼] │  ← ヘッダー
├────────────┬───────────────────────────────────────┤
│            │                                       │
│ ・ダッシュ │                                       │
│ ・ナレッジ │           メインエリア                │
│ ・SOP      │         （コンテンツ表示）            │
│ ・事故     │                                       │
│ ・相談     │                                       │
│ ・マイ     │                                       │
│            │                                       │
│            │                                       │
│  サイドバー│                                       │
└────────────┴───────────────────────────────────────┘
```

### 2.4 ダッシュボードの見方

| ウィジェット | 内容 |
|-------------|------|
| 重要な通知 | 最新の重要通知3件 |
| 承認待ち | 自分が提出したナレッジの状況 |
| 最近閲覧 | 直近5件のアクセス履歴 |
| よく見られている | 人気のナレッジ |
| 統計情報 | ナレッジ総数、今月の登録数 |

## 3. ナレッジ検索

### 3.1 基本検索

1. ヘッダーの検索バーにキーワード入力
2. Enterキーまたは虫眼鏡アイコンをクリック
3. 検索結果を確認

**例**: `コンクリート 養生` と入力

### 3.2 詳細検索

サイドバー「ナレッジ検索」→「詳細検索」

| 条件 | 使い方 |
|------|--------|
| キーワード | タイトル・本文を検索 |
| カテゴリ | SOP、法令、事故レポートなど |
| タグ | 関連キーワードで絞り込み |
| 優先度 | 高・中・低 |
| 日付範囲 | 作成日・更新日で絞り込み |

### 3.3 検索テクニック

| 検索方法 | 書き方 | 例 |
|---------|--------|-----|
| AND検索 | スペース区切り | `コンクリート 養生` |
| 完全一致 | `"フレーズ"` | `"打設後24時間"` |
| 除外 | `-キーワード` | `コンクリート -舗装` |

### 3.4 検索結果の活用

- **並び替え**: 関連度順、更新日順、閲覧数順
- **フィルタ**: ステータス、カテゴリで絞り込み
- **クリック**: 詳細画面を表示

## 4. ナレッジ作成

### 4.1 作成フロー

```
作成 → 下書き保存 → 承認申請 → 承認者確認 → 承認 → 公開
                        ↑               ↓
                        └── 却下（修正後再申請）
```

### 4.2 作成手順

1. サイドバー「ナレッジ検索」→「新規作成」
2. 必須項目を入力:
   - タイトル（わかりやすく）
   - 概要（200文字以内）
   - カテゴリ（SOP、法令、事故レポートなど）
   - 本文（Markdown対応）
3. タグを追加（複数可）
4. 添付ファイルをアップロード（任意）
5. 「下書き保存」または「承認申請」

### 4.3 良いナレッジの書き方

**タイトル**:
- ✅ `コンクリート打設時の夏期養生方法`
- ❌ `養生について`

**概要**:
- 何について書いているか
- どんな場面で役立つか

**本文構成例**:
```markdown
## 目的
この手順の目的を記載

## 適用範囲
どのような場合に適用するか

## 手順
1. 準備
2. 実施
3. 完了確認

## 注意事項
- 安全上の注意
- 品質上の注意

## 参考資料
- 関連SOP
- 参考法令
```

### 4.4 承認フロー

| 状態 | 説明 |
|------|------|
| 下書き | 自分だけが見える |
| 承認待ち | 承認者に通知が届く |
| 承認済み | 全ユーザーに公開 |
| 却下 | 却下理由を確認し修正 |

## 5. 通知と設定

### 5.1 通知の種類

| 通知タイプ | 内容 |
|-----------|------|
| 承認完了 | 自分のナレッジが承認された |
| 承認却下 | 却下理由を確認 |
| 新規SOP公開 | 新しいSOPが公開された |
| SOP改訂 | SOPが更新された |
| 専門家相談回答 | 相談に回答があった |
| 重要通知 | 緊急連絡 |

### 5.2 通知確認方法

1. ヘッダー右上のベルアイコン 🔔
2. 通知リストが表示
3. クリックで詳細画面へ

### 5.3 通知設定変更

1. ユーザーメニュー → 「設定」
2. 「通知設定」タブ
3. 受け取りたい通知をチェック
4. 「保存」

## 6. Q&A・演習

### よくある質問

**Q: パスワードを忘れた**
A: ログイン画面の「パスワードを忘れた場合」→メールでリセット

**Q: 検索しても見つからない**
A: キーワードを変える、詳細検索を使う、カテゴリを指定

**Q: 編集中に画面を閉じた**
A: 5分ごとに自動保存。マイページで下書きを確認

**Q: 却下されたナレッジは？**
A: 却下理由を確認し、修正して再申請

### 演習

1. **検索演習**: 「配筋検査」に関するナレッジを検索
2. **閲覧演習**: SOPを1つ開いて内容を確認
3. **作成演習**: 下書きでナレッジを1つ作成
