# 新機能監視ガイド

## 概要
新機能の監視方法について説明します。新機能は建設土木ナレッジシステムのナレッジ共有機能を指します。

## 監視対象
- APIエンドポイント: `/api/v1/new-feature/*`
- データベーステーブル: `new_feature_table`, `new_feature_logs`
- パフォーマンス指標: レスポンスタイム、CPU使用率、メモリ使用率

## 監視方法

### 1. ヘルスチェック
- スクリプト: `scripts/new-feature-health-check.sh`
- 実行頻度: 毎5分
- チェック項目:
  - APIエンドポイントの可用性
  - データベース接続
  - レスポンスタイム (< 3秒)

### 2. ログ監視
- ログファイル: `/var/log/mks/new-feature*.json`
- 監視パターン:
  - ERROR: エラーログ
  - WARNING: 警告ログ
  - INFO: 情報ログ

### 3. パフォーマンス監視
- 指標:
  - APIレスポンスタイム
  - データベースクエリ実行時間
  - リソース使用率

### 4. アラート設定
- Critical: APIダウン、DB接続失敗
- Warning: レスポンスタイム超過、ディスク容量90%超
- Info: 定期レポート

## 監視ツール
- Prometheus + Grafana (推奨)
- Zabbix (代替)
- スクリプトベース監視

## レポート
- 毎日のヘルスチェックレポート
- 毎週のパフォーマンスレポート
- 月次障害分析レポート
