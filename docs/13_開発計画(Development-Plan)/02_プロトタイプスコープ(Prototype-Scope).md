# プロトタイプスコープ

## 目的
- 現場判断を支援するための情報集約と可視化の有効性を検証する
- 承認・配信のワークフローが理解しやすいかを検証する
- 主要ユースケースの体験を通じて、情報優先度と画面構成を確認する

## 対象ユーザー
- 施工管理
- 品質保証
- 安全衛生

## 検証対象の課題
- ナレッジが分散し、検索・参照に時間がかかる
- 承認状況の把握が困難で、配信の遅れが発生する
- 事故・ヒヤリの再発防止策が現場間で共有されにくい

## インスコープ
- ダッシュボードの情報構成と優先順位
- 検索結果詳細/SOP/法令/事故/相談の詳細画面
- 承認フローと配信状況の可視化
- 主要インタラクション (タブ、フィルタ、進捗表示)

## アウトオブスコープ
- API連携や認証機能の実装
- 既存システムとのデータ同期
- 実運用を想定した権限設定の実装

## 前提・制約
- 静的HTML/CSS/JSで実装
- Docker等の仮想環境は使用しない
- 期間は未規定

## 成功条件 (フェーズ1)
- 対象ユースケースが明確に定義されている
- 画面範囲と検証課題が合意されている
