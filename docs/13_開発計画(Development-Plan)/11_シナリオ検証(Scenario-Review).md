# シナリオ検証・レビュー評価

## 目的
- プロトタイプが主要ユースケースを満たすかを数値評価する

## 評価基準
- 完成度 / 一貫性 / 実現性 / リスク / 受入準備
- 1〜5点で評価し、合計25点
- 20点以上を合格の目安とする

## 所見
- 主要導線は3クリック以内で到達でき、UIの一貫性も確保できている
- 承認フローの「次の担当」表示が明確で、誤解を生みにくい
- 相談起案の入力欄は必要情報を網羅している

## 改善候補
- 検索条件の状態が画面遷移後も保持されるUIが必要
- 詳細画面で関連ナレッジへの導線をより強調する
- KPIや進捗の更新タイミングを注記で明確化
