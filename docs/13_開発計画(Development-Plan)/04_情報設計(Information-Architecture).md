# 情報設計 (IA)

## 目的
- 画面内の情報優先度と配置ルールを明確化し、現場判断に必要な情報が最短で把握できる構造を作る

## 画面構成の原則
- 左から「全体状況 → 主要判断 → 補助情報」の順に配置
- 重要度の高い情報は上段/左寄せで表示
- 承認・配信の状態は常に可視化

## 画面レベルの情報階層
### ダッシュボード (index.html)
- 上段: 現場全体状況 (ヘッダー/主要KPI)
- 中段: 承認フロー、アクション、稼働モニタリング
- 下段: ナレッジフロー、法令チェック
- 右レール: 緊急アラート、現場ステータス、通知フィード

### 詳細画面 (検索/SOP/法令/事故/相談)
- 上段: 対象ナレッジのサマリ/メタ情報
- 中段: 詳細内容 (手順/分析/タスク)
- 右レール: 関連現場、レビュー状況、関連資料

## 情報ブロック定義
- 状態: 承認/配信/レビュー
- 判断: 施工判断ポイント、再発防止策
- 進捗: 進行率、SLA/稼働率
- 関連: 関連SOP/法令/過去事例

## 表示優先度 (高 → 低)
1. 緊急アラート/期限
2. 承認ステータス
3. 主要判断ポイント
4. 関連資料/補足情報

## 成功条件
- 主要判断情報が1画面内で把握できる
- 承認・配信の状態が視認できる
