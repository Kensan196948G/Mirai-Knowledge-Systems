# Phase B-6/B-7 残タスク確定

## 目的
B-6（検索・通知）およびB-7（WebUI統合）の残タスクを確定し、実装/受入基準を明確化する。

## B-6: 検索・通知機能（残タスク）

### 検索強化
- [ ] 全文検索の導入方針を確定（Meilisearch / PostgreSQL FTS / 現行JSON強化）
- [ ] 検索APIの拡張（ページネーション/ソート/フィルタ拡張）
- [ ] 検索条件の保持・履歴
- [ ] 関連ナレッジの自動提示（簡易版）

**受入基準**
- 主要一覧で `page/per_page/sort/order` が利用できる
- `search` クエリでタイトル/概要/本文を対象に検索できる
- 検索条件がリロード後も保持される

### 通知配信
- [ ] 通知モデルの確定（内部通知JSON + 将来の外部連携）
- [ ] 通知APIの追加（一覧/既読/配信）
- [ ] 重要イベントの通知トリガー定義（承認/却下/更新）

**受入基準**
- 主要イベントで通知が生成される
- 通知の一覧・既読更新がAPIで動作する

## B-7: WebUI統合実装（残タスク）

### 管理画面の認証対応
- [ ] `admin.html` をJWT認証前提に統合（`fetchAPI` 利用）
- [ ] 未認証時のログイン画面リダイレクト
- [ ] APIベースURLを `/api/v1` に統一

### 権限UI
- [ ] 権限に応じたUI表示制御（ボタン/操作の非表示）
- [ ] 権限不足時の表示メッセージ整理

**受入基準**
- `admin` 以外は管理画面の主要操作が不可（UI/API両面）
- 認証切れ時は自動的にログインへ遷移
