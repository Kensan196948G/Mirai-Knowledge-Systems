# UI方針 詳細

## 目的
- 現場判断に必要な情報を短時間で把握できる視認性を担保する
- 画面間の一貫性を維持し、学習コストを下げる

## デザイン原則
- 優先順位の可視化: 重要度は色と配置で表現する
- 情報密度の最適化: 1画面に必要情報を集約し、過不足を避ける
- 状態の明確化: 承認/配信/注意/危険を一目で把握

## タイポグラフィ
- 見出し: 明朝系で強調 (例: `.detail-title`)
- 本文: ゴシック系で可読性を確保
- 情報メタ: 12px前後で補助情報として配置

## 色の使い分け
- 通常情報: `--steel` / `--muted`
- 注意: `--warning`
- 危険: `--danger`
- 承認/完了: `--moss`
- 重要操作: `--accent`

## 余白・レイアウト
- 基本パネルは16〜24pxのパディング
- カード間隔は10〜16pxで統一
- 3カラム構成を基本とし、狭幅では1カラムに集約

## 状態表現
- 承認状態: `.badge` の色で区分
- リスク状態: `.status-dot` で即時判断
- 進捗: `.progress-track` と `.progress-fill` で視覚化

## インタラクション
- フィルタ/タブは即時切替
- 重要ボタンはCTA色を使用
- 変化は0.8s以内の緩やかなアニメーション

## アクセシビリティ
- 文字色と背景のコントラストを確保
- 重要情報は色だけでなくラベルも併記
- `prefers-reduced-motion` に配慮

## 適用対象
- `webui` の全画面
