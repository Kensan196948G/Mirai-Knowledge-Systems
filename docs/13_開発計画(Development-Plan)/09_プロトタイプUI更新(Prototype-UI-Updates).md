# プロトタイプUI更新

## 目的
- UI方針詳細とコンポーネント仕様に合わせて画面の視認性と一貫性を高める

## 実施内容
- フォーカス表示の明確化 (視認性/操作性の向上)
- テーブルのモバイル対応 (横スクロールの許容)
- タブ/フィルタのアクセシビリティ属性の整備

## 実装差分
- `webui/styles.css`
  - `:focus-visible` のアウトライン追加
  - 小画面での `.table` の横スクロール対応
- `webui/index.html`
  - タブに `role`/`aria` を追加
  - フィルタに `aria-pressed` を付与

## 成果物
- `webui/index.html`
- `webui/styles.css`

## レビュー評価観点
- デザインの統一感
- 文字の可読性
- 主要操作の分かりやすさ
